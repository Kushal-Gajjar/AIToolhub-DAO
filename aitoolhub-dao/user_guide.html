1. Project Overview & Tech Stack

AIToolHub DAO is a decentralized autonomous organization governing a marketplace of heavy AI utilities. DAO participants vote on budgets, propose tool specifications, and stake governance tokens to claim a share of platform revenues. The project utilizes a decoupled containerized environment divided into distinct service responsibilities:

Scope Technologies Responsibility
Frontend HTML5, CSS3, Vanilla JS, Nginx Renders dashboards, charts, governance states, and profile portals.
Backend API C#, ASP.NET Core 8, EF Core, PostgreSQL Stores user auth accounts, proposals, tool lists, and casts.
AI Services Python, FastAPI, Celery, Redis Runs transformer pipelines, OCR processing, and zero-shot NLP models.
Contracts Solidity, Hardhat, OpenZeppelin Governor Enforces voting periods, quorum metrics, and delegation weights on-chain.

2. Directory Structure Blueprint

The code is organized into a nested hierarchy mapping directly to service images within the orchestrator configuration:

├── aitoolhub-frontend/      # Frontend Client Directory (served by Nginx container)

│   │   ├── index.html       # Single entry-point HTML document
│   │   ├── app.js           # Client-side javascript state and handlers
│   │   └── styles.css       # Layout styles and custom theme parameters

│   ├── Controllers/         # Auth, Tools, Proposals, and Votes controllers
│   ├── Models/              # Database entities (User, Proposal, Vote, Tool)
│   ├── Services/            # Business operations implementations
│   └── Program.cs           # Web host startup configurations & integrations
│
├── aitoolhub-ai/            # Python Inference Engine
│   ├── services/            # Pipelines (ocr_service, summarizer_service, etc.)
│   ├── main.py              # FastAPI controller definitions
│   └── celery_worker.py     # Background worker consuming broker queues
│
├── aitoolhub-contracts/     # Smart Contracts Development Environment
│   └── contracts/           # DAOGovernance.sol OpenZeppelin token governor
│
└── docker-compose.yml       # Coordinates redis, db, api, ai, workers, and frontend services

4. Startup & Execution Steps

Start the platform containers using Docker Compose. Ensure Docker Desktop is active before proceeding:

💡 Important Network Hint

On the first run or when rebuilds are initiated, reload Nginx once containers are up to ensure proper upstream DNS sync.

# 1. Navigate to directory
cd aitoolhub-dao

# 2. Compile images and start containers in detached mode
docker-compose up --build -d

# 3. Synchronize Nginx upstream proxy resolution
docker-compose restart nginx

Once active, verify URLs:

Web Service Hosting Address Developer Resource
Frontend Client http://localhost:3000 or http://localhost Production UI View
C# Backend API http://localhost:5000/swagger Interactive OpenAPI Specification Swagger UI
FastAPI AI Server http://localhost:8000/docs Swagger documentation for ML models

5. Relational Configurations (.env)

Duplicate `.env.example` to `.env` and adjust the variables to reflect local parameters:

# PostgreSQL connection credentials
POSTGRES_DB=aitoolhub
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres_password_123

# C# REST Host Database String
ConnectionStrings__DefaultConnection=Host=db;Database=aitoolhub;Username=postgres;Password=postgres_password_123

# JWT Token validation secret (Must be at least 32 characters long)
Jwt__Secret=YOUR_SECURE_JWT_SIGNING_PASSPHRASE_KEY_32_CHARS
Jwt__Issuer=aitoolhub-api
Jwt__Audience=aitoolhub-frontend

# Redis caching url link
REDIS_URL=redis://redis:6379/0

6. Developer Operations (Hot-Deploy)

Nginx serves static files dynamically. When adjusting HTML, CSS, or JS layouts on the client side, hot-deploy updates immediately into the container without rebuilding the Docker image:

# Deploy updated HTML layout
docker cp aitoolhub-frontend/public/index.html aitoolhub-dao-frontend-1:/usr/share/nginx/html/index.html

# Deploy updated JavaScript engine
docker cp aitoolhub-frontend/public/app.js aitoolhub-dao-frontend-1:/usr/share/nginx/html/app.js

# Deploy updated styling layouts
docker cp aitoolhub-frontend/public/styles.css aitoolhub-dao-frontend-1:/usr/share/nginx/html/styles.css

# Force client reload inside proxy
docker exec aitoolhub-dao-nginx-1 nginx -s reload

7. Troubleshooting & Logs

Consult this checklist when facing environment initialization failures:

1. "Cannot connect to the Docker daemon" error:
Make sure Docker Desktop application is launched and running in the taskbar.

2. Swagger returns 502 Bad Gateway:
PostgreSQL database migrations are applied on start. Give the container 15-30 seconds to complete migrations before checking Swagger docs.

3. Relational status queries failed / unreachable warnings:
Inspect logs for a specific container to discover errors: docker-compose logs api
